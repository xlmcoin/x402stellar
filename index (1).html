Protocol Flow

How x402 + Stellar works

An AI agent sends an HTTP request. The server responds with 402. The agent pays with x402 on Stellar and retries. That's the entire flow.

STEP 01
Agent sends HTTP request
No account, no API key, no session. Just a standard HTTP GET or POST to any x402-protected endpoint.
GET /api/data HTTP/1.1 Host: api.example.com
STEP 02
Server responds 402
The server returns HTTP 402 with a payment payload describing the price and accepted Stellar asset — x402 token.
HTTP/1.1 402 Payment Required X-Payment-Required: { "asset": "x402:G...", "amount": "1.00" }
STEP 03
Agent pays and retries
The client settles payment on Stellar in under 5 seconds, attaches proof to the request header, and retries — access granted.
GET /api/data HTTP/1.1 X-Payment: <stellar-tx-hash>
Developer Integration

Accept x402 payments
on Stellar in minutes

One middleware call is all it takes. x402 handles the 402 response, payment verification, and request retry automatically.

// Install: npm install @x402/express stellar-sdk import { paymentMiddleware } from '@x402/express'; import { Asset } from 'stellar-sdk'; // x402 token on Stellar const X402_ASSET = new Asset( 'x402', 'GD6QKNCVIVYG7IWY2Q7JLDOCT5S73KOH3NDV4YMQBFDMH7BRXKB6BOSS' ); app.use(paymentMiddleware({ "GET /api/weather": { accepts: [{ network: 'stellar', asset: X402_ASSET, maxAmountRequired: '1.0000000', }], description: "Live weather data — paid per request", }, })); // That's it. Stellar settles. Your API earns.

FAQ

Common questions

What is x402?
x402 is the open payment standard built around the HTTP 402 Payment Required status code. It lets services charge for API access and content directly over HTTP — without accounts, sessions, or credential management. Both sellers and buyers interact through standard HTTP requests, with payment handled transparently through the protocol.
Why Stellar?
Stellar offers near-instant finality (under 5 seconds), negligible transaction fees, and a native asset issuance mechanism ideal for tokens like x402. Its built-in decentralized exchange also makes x402 tradeable without centralized intermediaries — perfectly aligned with x402's zero-centralization philosophy.
How do I add the x402 trustline in my Stellar wallet?
In any Stellar wallet (Lobstr, Solar, or XBULL), go to "Add Asset" and enter the asset code x402 with issuer address GD6QKNCVIVYG7IWY2Q7JLDOCT5S73KOH3NDV4YMQBFDMH7BRXKB6BOSS. Confirm the trustline transaction (costs ~0.5 XLM in reserve). You can then receive, send, and trade x402.
Is x402 production-ready?
The x402 protocol is open-source under the Apache-2.0 license and actively maintained. Refer to the official documentation at docs.x402.org for the current stability status and integration guides. The Stellar token is live on mainnet.
Are there any protocol fees?
Zero protocol fees. x402 is a neutral standard — sellers and buyers pay only the nominal Stellar network fee per transaction (fractions of a cent). No middleman, no rent extraction.
What is the total supply of x402?
The maximum supply of x402 tokens on Stellar is 6,000,000,000 (6 billion). This is fixed at issuance and verifiable on-chain via the issuer address GD6QKNCVIVYG7IWY2Q7JLDOCT5S73KOH3NDV4YMQBFDMH7BRXKB6BOSS.
